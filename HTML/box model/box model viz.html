Two type element based pn layout rules.

1.block level.
display : block、table、flex、grid and list-item.

By default, the width is 100% and height is 0. You can set width、padding、margin和border

2.inline-level.
display : inline, inline-block, inline-table, and inline-flex, inline-grid

There can be many elements in one line, arranged from left to right. You can't set width, height, margin-top and margin-bottom. By default, the alignment is left(horizontally) and baseline(vertically). The alignment is determined by the 'text-align' of the parent and 'vertical-align' of itself.

The space between each inline-level element is caused by the enter / space between them.

2-1. inline-block-level. For the inline-block-level element, you can set aby size just like block level element.
However, by default, the width is 0. I set the height of span element as 100px, but it just affect the span here since I set display to inline-block;

Formatting Context has specific rules of layout.

BFC: block formatting context. There 4 main properties that can be used to create a BFC:

  • float: not none
  • position: absolute or fixed
  • display: flex, grid, table-cell, table-caption
  • overflow: not visible

There are 3 main rules

  1. Children elements' css style won't affect elements outside the BFC parent element.
  2. Floating children elements are invloved in the calculation of the height of the parent.
  3. BFC elements in the normal document flow can recognize floating, sibling elements.
